# Contributing to BoilerHAUS Ops

Thanks for your interest in contributing to `boilerhaus-ops`. This document outlines how to get involved, submit changes, and maintain quality across the repository.

## How to Contribute

1. **Check open issues** — Look at [Issues](https://github.com/BoilerHAUS/boilerhaus-ops/issues) for things labeled `priority:now` or `good first issue`.
2. **Fork or branch** — Create a feature branch from `main` (e.g., `feat/your-feature` or `fix/your-fix`).
3. **Make your changes** — Keep commits focused and well-described.
4. **Open a PR** — Reference the issue number in the PR description (e.g., `Closes #21`).
5. **Respond to review** — Address feedback promptly; iterate until approved.

## Branching and PR Workflow

- Branch naming: `feat/<topic>`, `fix/<topic>`, `docs/<topic>`, `chore/<topic>`
- PRs should target `main` unless otherwise specified.
- Squash or rebase before merge when the commit history is noisy.
- Every PR should reference at least one issue.

## Code and Docs Style

- Follow the [STYLE_GUIDE.md](docs/STYLE_GUIDE.md) (when available) for documentation formatting, voice, and structure.
- Use Markdown for all docs. Prefer Mermaid for diagrams.
- Keep files focused — one topic per document.

## Testing and CI

- Ensure any CI checks pass before requesting review.
- If your change introduces a new process or document, update the README or relevant index.

## Issue Templates

- Use the provided issue templates in `.github/ISSUE_TEMPLATE/` when filing new issues.
- Label issues with `priority:now`, `priority:next`, or `priority:later` as appropriate.

## Questions

If you're unsure about anything, open an issue with the `question` label or reach out in the relevant discussion thread.
