# GitHub Workflow

Use docs to define work before creating issues.

## Workflow

1. Update or create the relevant source-of-truth document in `docs/`.
2. Place the work in the right milestone in [milestones.md](./milestones.md).
3. Add or update the work item in [issue-map.md](./issue-map.md).
4. Open a focused GitHub issue using the template in `.github/ISSUE_TEMPLATE/work-item.md`.
5. Keep one outcome per issue whenever possible.

## Rules

- prefer one business outcome per issue
- link issues back to the doc they implement or update
- close or rewrite stale issues instead of letting them drift
- when a decision changes, update the doc and the issue backlog in the same pass
