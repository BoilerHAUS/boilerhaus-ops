# Incident Response Playbook

## Purpose
Incident response playbook for boilerhaus-ops incidents.

## Roles
- **Commander:** Leads the response and makes final calls
- **Comms:** Handles external and internal stakeholder communication
- **Scribe:** Documents the incident timeline and decisions

## Trigger Phrases
- "Production is down"
- "Security breach suspected"
- "Financial data exposed"

## Playbook Steps

1. **Acknowledge** — Confirm the incident is real and document it
2. **Assess** — Determine severity (P0 = immediate, P1 = fix today, P2 = fix this week)
3. **Contain** — Stop the bleed. Isolate the affected system
4. **Resolve** — Fix the root cause
5. **Communicate** — Update stakeholders on what happened
6. **Review** — Schedule a postmortem within 48 hours
