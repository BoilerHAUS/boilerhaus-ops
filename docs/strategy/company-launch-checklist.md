# Boilerhaus Company Launch Checklist

## Purpose

Define what must be true before Boilerhaus is treated as launch-ready as a real operating company.

This checklist is company-first.
It is broader than product beta readiness.

## 1. Legal formation

- [ ] legal name decision is documented
- [ ] fallback name or numbered-corporation plan is documented
- [ ] federal incorporation is filed and accepted
- [ ] certificate and articles are saved in the records folder
- [ ] bylaws are adopted
- [ ] shares are issued
- [ ] officers are appointed
- [ ] ISC register is created and filed
- [ ] Ontario business-name or registry filings needed for `ScopeHouse` are completed or explicitly ruled out

## 2. CRA and tax setup

- [ ] business number access is added to CRA My Business Account
- [ ] GST HST decision is documented
- [ ] payroll decision is documented
- [ ] corporate year-end and tax support path are documented

## 3. Banking and finance

- [ ] business bank account is open
- [ ] signing authority is documented
- [ ] founder reimbursement workflow is documented
- [ ] bookkeeping system is chosen
- [ ] first chart of accounts exists
- [ ] invoice template exists
- [ ] proposal template exists
- [ ] payment collection path is tested

## 4. Operating system

- [ ] this repo is the source of truth for operating docs
- [ ] Nextcloud or the chosen file system has a live folder structure
- [ ] legal, finance, customer, and operations folders exist
- [ ] primary email inboxes and aliases are documented
- [ ] password manager is chosen
- [ ] 2FA is enabled on critical systems
- [ ] account inventory exists with owner and recovery path

## 5. Go-to-market basics

- [ ] founder-led service offer is documented
- [ ] pricing hypotheses are documented
- [ ] discovery script is ready
- [ ] demo script is ready
- [ ] lead tracker is live
- [ ] first target account list exists
- [ ] public contact path exists
- [ ] minimum website and legal pages are identified

## 6. Operating cadence

- [ ] weekly business review owner is named
- [ ] weekly review schedule exists
- [ ] minimum dashboard fields are defined
- [ ] top launch blockers are visible in GitHub issues
- [ ] the first weekly review has been run once

## Launch gate

Boilerhaus is launch-ready when:

- the company exists legally and the records are complete
- money can be invoiced, collected, booked, and reconciled
- critical accounts are secure and recoverable
- the first sales motion is documented and runnable
- the business is being reviewed weekly against real metrics and blockers
