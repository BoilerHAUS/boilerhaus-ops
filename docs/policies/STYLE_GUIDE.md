# Style Guide

## Purpose
Establish writing style for boilerhaus-ops docs.

## Voice and Tone
- Direct and professional
- Use active voice
- Prefer short, declarative sentences
- Write for a reader who is smart but context-poor

## Formatting
- Use title case for headers
- Code snippets get inline backticks for single words, fenced blocks for more
- Dates: ISO 8601 (2026-04-08)
- Lists: use bullets for unordered, numbers for sequential

## Architecture Diagrams
- Use Mermaid for sequence and flow diagrams
- Use plain-ASCII for simple topology maps

## Terminology
- Boilerhaus — the legal entity and holding company
- ScopeHouse — the first product and primary public-facing name
- boilerhaus-ops — this repository, the ops control plane
